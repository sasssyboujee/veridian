# Veridian - Off-Chain Oracle & Yield Engine

This directory contains the Python FastAPI backend for the Veridian Real-World Asset (RWA) Escrow Platform. 

It serves as the critical bridge between physical hardware in the real world and the smart contracts on the blockchain. It is responsible for ingesting hardware data, cryptographically verifying its origin, crunching the yield math, and preparing the exact oracle payloads for the chain.

## 🧠 Core Responsibilities

1. **Telemetry Ingestion & Security (`/telemetry`)**: 
   Accepts IIoT data from deployed physical assets. Uses ECDSA signatures to verify that the data was exclusively generated by a hardware Root-of-Trust (TPM 2.0 or Secure Enclave). If an operator tries to falsify data with an invalid signature, the system can flag it to slash their bond.
2. **Dynamic Yield Engine (`services/yield_engine.py`)**: 
   When a billing cycle ends, it calculates the gross yield (e.g. 100 kWh generated * $0.16 rate). It then deterministically splits this into the 75/8/7/5/5 fee waterfall.
3. **Chainlink Oracle Payload Generation (`/yields/oracle/{asset_id}`)**: 
   Once calculations are finalized, it generates a strict JSON payload that Chainlink Functions nodes can fetch. The payload scales the yield into 18-decimal `wei` so the smart contract can execute the distribution flawlessly.

## 🛠️ Tech Stack

*   **Framework**: FastAPI (Python)
*   **Database**: PostgreSQL (Supabase) accessed via SQLAlchemy Async ORM
*   **Cryptography**: `cryptography` library for ECDSA SECP256R1 signature verification.

## 🚀 Getting Started

1. Ensure your `.env` is configured with your Supabase `DATABASE_URL` (using port 6543 for transaction pooling).
2. Set up the virtual environment and install dependencies:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```
3. Run the development server:
   ```bash
   uvicorn app.main:app --reload --port 8000
   ```

## 🧪 Simulation Scripts

We provide scripts to easily populate data and test the flow without needing physical hardware:

*   **`seed_demo_data.py`**: Wipes the local database and inserts 12 diverse assets (Solar, Wind, Agriculture) along with 6 months of perfectly calculated historical telemetry and yield history. Essential for preparing the Admin dashboard for a live demo.
*   **`simulate_flow.py`**: Runs an end-to-end integration test creating a single asset, ingesting data, and calculating yield instantly to verify the math logic.
